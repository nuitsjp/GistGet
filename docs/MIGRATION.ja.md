# 移行ガイド

## 概要
このドキュメントでは、PowerShell 版 GistGet から新しい C# 版への移行手順について説明します。

## 互換性
C# 版は、PowerShell 版で使用されていた `packages.yaml` 形式と完全な互換性を持つように設計されています。既存の Gist ファイルを変更する必要はありません。

## 移行手順

1.  **新しい GistGet のインストール**
    (インストール手順は後日追加予定)

2.  **認証**
    `gistget auth login` を実行して、GitHub アカウントで認証します。これは、PowerShell 版で使用されていた Personal Access Token (PAT) 方式に代わるものです。

3.  **設定の確認**
    `gistget sync --dry-run` (利用可能な場合) を実行するか、Gist の内容を確認してください。

4.  **PowerShell 版のアンインストール**
    コマンドの競合を避けるため、不要になった PowerShell モジュールまたはエイリアスを削除してください。

## 主な変更点
-   **パフォーマンス**: C# 版は大幅に高速化され、堅牢性が向上しています。
-   **認証**: 生の PAT の代わりに、安全な Device Flow を使用します。
-   **出力**: 出力フォーマットと進捗表示が改善されました。
-   **エラー処理**: エラー報告と復元力が向上しました。

## トラブルシューティング
移行中に問題が発生した場合は、`KNOWN_ISSUES.ja.md` を確認するか、GitHub で問題を報告してください。
